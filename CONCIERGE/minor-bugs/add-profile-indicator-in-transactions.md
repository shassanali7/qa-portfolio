## Bug: Add Profile Indicator in Transactions

# Module
Transaction Page

# Platform
Web

# Description
Transactions do not indicate whether they belong to the main profile or a sub-profile, making it difficult for users to identify the source of each transaction.

# Steps to Reproduce
1. Login to the application
2. Navigate to Transaction Page
3. Observe the list of transactions

# Expected Result
- Each transaction should clearly indicate whether it belongs to:
1. Main profile
2. Sub-profile

# Actual Result
- No distinction between main and sub-profile transactions
- All transactions appear similar

# Severity
Minor

# Priority
Medium

# Impact
- Confusing transaction tracking
- Poor user experience

# Notes
Suggested to add label, tag, or column indicating profile type
