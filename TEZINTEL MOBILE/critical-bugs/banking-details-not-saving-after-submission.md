# Bug: Banking Details Not Saved After Submission

## Module
Group → Banking Details

## Platform
Mobile App

## Description
Banking details entered by the user are not saved after submission.

## Steps to Reproduce
1. Navigate to Banking Details section  
2. Enter valid banking information  
3. Submit the form  

## Expected Result
- Banking details should be saved successfully  
- Data should persist on revisit  

## Actual Result
- Data is not saved  
- Fields appear empty after submission  

## Severity
Critical

## Priority
Critical

## Impact
- Prevents financial operations  
- Breaks trust in system reliability  

## Notes
Likely issue with API integration or database persistence
