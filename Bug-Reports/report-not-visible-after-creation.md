# Bug: Newly Created Report Not Visible in List

## Module
Reports

## Platform
- iOS

## Description
After creating a report, it does not appear in the report listing unless the user manually searches or refreshes the page.

## Steps to Reproduce
1. Create a new report  
2. Navigate to the report listing page  

## Expected Result
- Newly created report should appear immediately in the list  

## Actual Result
- Report is not visible  
- Appears only after refresh or search  

## Severity
High

## Priority
High

## Impact
- Users think the report was not created  
- Causes confusion and duplicate submissions  

## Notes
Possible issue with list refresh, caching, or API sync delay
