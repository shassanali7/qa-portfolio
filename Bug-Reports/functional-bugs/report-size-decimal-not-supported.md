# Bug: Decimal Values Not Accepted in Size Field

## Module
Reports

## Platform
- Android

## Description
The Size input field in the Reports module does not accept floating-point (decimal) values. This prevents users from entering accurate measurements.

## Steps to Reproduce
1. Navigate to Reports  
2. Start filling report details  
3. Enter decimal value (e.g., 10.5) in Size field  

## Expected Result
- Field should accept decimal values  

## Actual Result
- Decimal input is not allowed  

## Severity
Medium

## Priority
Medium

## Impact
- Limits accurate data entry  
- Affects report precision  

## Notes
Input validation may be restricting numeric format
