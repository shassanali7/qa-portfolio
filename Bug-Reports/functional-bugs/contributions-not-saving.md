# Bug: Contribution Details Not Saved (Internal & External)

## Module
Group → Contributions

## Platform
Mobile App

## Description
Contribution details are not being saved after submission for both Internal and External contributions. This affects all contribution types including one-time, installment, collect, and payout.

## Steps to Reproduce
1. Navigate to Group → Contributions  
2. Select Internal → Click "+ Add"  
3. Enter details (One-time / Installment)  
4. Submit  

Repeat same for External → Collect / Payout  

## Expected Result
- Contribution details should be saved successfully  
- Data should reflect in contribution list  

## Actual Result
- No data is saved after submission  
- No confirmation or error shown  

## Severity
Medium

## Priority
Medium

## Impact
- Core functionality broken  
- Financial tracking becomes unusable  
- Blocks user workflow completely  

## Notes
Possible backend/API failure or missing state update
