## Bug: Transactions Not Visible by Default

# Module
Transactions

# Platform
iOS

# Description
The transaction page does not display any records by default. Users must apply filters to view transactions, which creates confusion and misleading empty state.

# Steps to Reproduce
1. Login to the app
2. Navigate to Transactions page

# Expected Result
* *All transactions should be displayed by default
* Actual Result
* No transactions shown initially
* Records only appear after applying filters

# Severity
Critical

# Priority
High

# Impact
* Misleading UI behavior
* Users may assume no transactions exist

# Notes
Possible default filter or API query issue
